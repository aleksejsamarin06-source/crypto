# CryptoSafe Manager

CryptoSafe Manager - локальный менеджер паролей с графическим интерфейсом на PySide6. Хранилище находится в выбранном пользователем SQLite-файле `.db`, а записи шифруются на стороне клиента.

Текущая версия: **0.8.0 (Sprint 8)**.

## Возможности

- Создание и открытие отдельных файлов хранилища `.db`.
- Защита мастер-паролем.
- Шифрование записей через AES-GCM.
- Добавление, редактирование, удаление и поиск записей.
- Категории записей и предпросмотр при импорте/экспорте.
- Генератор паролей и проверка надёжности мастер-пароля при создании БД.
- Кнопка показа/скрытия пароля в мастере первого запуска.
- Безопасный буфер обмена с автоочисткой.
- Блокировка хранилища при сворачивании, Alt+Tab или переходе в другое приложение.
- Отложенная блокировка по таймеру из настроек.
- Разблокировка последней открытой БД через окно мастер-пароля без повторного выбора файла.
- Иконка в системном трее с состояниями заблокировано/открыто/panic mode.
- Настройка темы интерфейса: тёмная, светлая или системная; по умолчанию используется тёмная тема.
- Panic mode по `Ctrl+Shift+Q` и через меню трея.
- Журнал аудита, проверка целостности и экспорт логов.
- Импорт и экспорт: encrypted JSON, CSV, Bitwarden JSON, LastPass CSV.
- Шифрование экспорта паролем или публичным `.pem` ключом получателя.
- Импорт encrypted JSON по паролю или приватному `.pem` ключу.
- Sharing-пакеты и визуальный QR-код со сроком действия в минутах.
- Приём shared package через `Файл -> Принять запись`.
- Pytest-отчёт и coverage report в `tests/report/`.
- PyInstaller-сборка для запуска двойным кликом.

## Установка из исходников

```powershell
git clone https://github.com/aleksejsamarin06-source/crypto.git
cd crypto
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
```

## Запуск из исходников

```powershell
python run.py
```

Также можно:

```powershell
python main.py
```

## Запуск как обычное приложение

Собрать приложение:

```powershell
python -m PyInstaller CryptoSafeManager.spec --noconfirm
```

После сборки запускать:

```text
dist\CryptoSafeManager\CryptoSafeManager.exe
```

Если нужно отправить приложение другому человеку, архивируйте всю папку:

```text
dist\CryptoSafeManager
```

Один `.exe` отдельно отправлять не нужно: рядом лежат библиотеки PySide6 и зависимости.

## Первый запуск

1. Создайте новую базу через мастер первого запуска или меню `Файл -> Новый`.
2. Введите мастер-пароль.
3. При необходимости измените имя файла БД. Значение по умолчанию `CryptoSafe.db` уже подходит для создания.
4. Завершите мастер.
5. Новая база сразу остаётся открытой, повторно выбирать файл не нужно.

Требования к мастер-паролю:

- минимум 8 символов;
- хотя бы одна заглавная буква;
- хотя бы одна цифра;
- спецсимвол рекомендуется, но не обязателен.

## Блокировка

В настройках можно выбрать режим блокировки:

- отключена;
- сразу;
- через заданное время.

Блокировка срабатывает при сворачивании окна, Alt+Tab или клике в другое приложение. Если включено сворачивание в трей, приложение скрывается в трей. При возвращении в окно показывается ввод мастер-пароля для последней открытой БД.

## Тема

В настройках доступен выбор темы:

- тёмная;
- светлая;
- как в системе.

По умолчанию включена тёмная тема, чтобы приложение одинаково выглядело на Windows 10 и Windows 11. В системном режиме приложение пытается определить тему Windows через настройки пользователя.

## Импорт и экспорт

Экспорт поддерживает:

- encrypted JSON;
- CSV;
- Bitwarden JSON.

Импорт поддерживает:

- encrypted JSON;
- CSV;
- Bitwarden JSON;
- LastPass CSV.

Для encrypted JSON можно использовать пароль или `.pem` ключ. В окнах импорта и экспорта есть предпросмотр записей по категориям.

## Приём записи

Чтобы принять запись, созданную через `Поделиться записью`:

1. Откройте свою БД.
2. Выберите `Файл -> Принять запись`.
3. Выберите JSON-файл shared package.
4. Введите пароль пакета или выберите приватный `.pem` ключ, если пакет был зашифрован публичным ключом.
5. Запись будет добавлена в текущее хранилище.

## Журнал аудита

В журнале действий доступна кнопка `Проверить целостность`. Проверка смотрит хеш-цепочку записей и криптографические подписи журнала. Если отчёт показывает `Целостность: ПОДТВЕРЖДЕНА`, повреждения журнала не обнаружены.

Показатель `Подписи верны` считается только по тем записям, у которых есть подпись и доступен мастер-пароль текущей БД. Старые записи без подписи отдельно отображаются как `Записей без подписи`.
