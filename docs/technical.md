# Техническая документация CryptoSafe Manager

## Назначение

CryptoSafe Manager - локальный менеджер паролей на Python и PySide6. Приложение хранит данные в SQLite-файле `.db`, который выбирает пользователь, и шифрует записи на стороне клиента.

## Основные модули

- `main.py` и `run.py` - точки входа.
- `src/gui/main_window.py` - главное окно, меню, трей, блокировка, panic mode.
- `src/gui/widgets/` - диалоги создания БД, входа, смены пароля, импорта, экспорта и sharing.
- `src/database/db.py` - создание таблиц и базовые операции SQLite.
- `src/core/vault/` - работа с зашифрованными записями.
- `src/core/crypto/` - derivation ключей, проверка мастер-пароля и key storage.
- `src/core/import_export/` - импорт, экспорт, public-key encryption и sharing.
- `src/core/audit/` - журнал аудита, подписи и проверка целостности.
- `src/core/security/` - activity monitor, panic mode, memory guard и side-channel helpers.
- `tests/` - автоматические тесты.

## База данных

Основные таблицы:

- `vault_entries` - зашифрованные записи.
- `settings` - настройки приложения и безопасности.
- `key_store` - auth hash, salts и параметры derivation.
- `audit_log` - журнал аудита.
- `shared_entries` - данные sharing-пакетов.
- `import_export_history` - история импорта и экспорта.
- `contacts` - публичные ключи получателей.

## Криптография

- Записи шифруются через AES-GCM.
- Мастер-пароль проверяется через Argon2id в модуле authentication/key derivation.
- Для совместимых ключей используется PBKDF2-HMAC-SHA256.
- Экспорт может шифроваться паролем или публичным RSA `.pem` ключом.
- Импорт encrypted JSON поддерживает пароль и приватный `.pem` ключ.
- Журнал аудита использует хеш-цепочку и подписи записей.

## Блокировка

Приложение поддерживает явные состояния хранилища:

- `LOCKED` - база закрыта, автоматического диалога разблокировки нет;
- `UNLOCKED` - база открыта и ключ находится в памяти;
- `PENDING_UNLOCK` - база закрыта после потери фокуса/сворачивания, диалог пароля будет показан при возврате окна;
- `PANIC` - panic mode.

Режимы блокировки:

- отключена;
- сразу;
- через заданное время.

Блокировка срабатывает при сворачивании окна, Alt+Tab или переходе в другое приложение. Если включён трей, окно скрывается в системный трей. При возврате показывается окно ввода мастер-пароля для последней открытой БД.

## Тема интерфейса

Тема хранится в настройке `app_theme`. Возможные значения:

- `dark`;
- `light`;
- `system`.

Значение по умолчанию - `dark`. Системный режим на Windows читает `HKCU\Software\Microsoft\Windows\CurrentVersion\Themes\Personalize\AppsUseLightTheme`. Если определить системную тему не удалось, используется тёмная тема.

## Panic Mode

Panic mode активируется через `Ctrl+Shift+Q` или меню трея. При активации очищается чувствительное состояние, блокируется хранилище, очищается буфер обмена и меняется состояние иконки в трее.

## Импорт и экспорт

Поддерживаемые форматы:

- encrypted JSON;
- CSV;
- Bitwarden JSON;
- LastPass CSV.

В окнах импорта и экспорта есть предпросмотр записей по категориям. Для экспорта можно выбрать отдельные записи и категории.

## Тестирование

Основной запуск:

```powershell
python -m pytest
```

Генерация отчёта:

```powershell
python scripts\generate_test_report.py
```

Отчёты сохраняются в `tests/report/`.

Текущий sprint8-результат:

- `55 passed`;
- `1 skipped`;
- coverage около `82%`.

## Сборка

Сборка приложения для запуска двойным кликом:

```powershell
python -m PyInstaller CryptoSafeManager.spec --noconfirm
```

Результат:

```text
dist\CryptoSafeManager\CryptoSafeManager.exe
```

Для передачи приложения другому человеку архивируется вся папка `dist\CryptoSafeManager`, а не только `.exe`.

## Ограничения

- Нет облачной синхронизации.
- Нет браузерного расширения.
- QR-код отображается визуально, но сканирование камерой внутри приложения не реализовано.
- PyInstaller-сборка ориентирована в первую очередь на Windows.
